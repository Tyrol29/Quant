# Week 1 知识要点整理

## 一、基础概念

### 1. 股票数据结构 (OHLCV)

| 字段 | 含义 | 说明 |
|------|------|------|
| `Open` | 开盘价 | 当日第一笔成交价 |
| `High` | 最高价 | 当日最高成交价 |
| `Low` | 最低价 | 当日最低成交价 |
| `Close` | 收盘价 | 当日最后一笔成交价 |
| `Volume` | 成交量 | 当日总成交手数（1手=100股）|

---

## 二、技术指标详解

### 1. 极差 (Range)

**定义**：当日最高价与最低价的差值

**公式**：
```
Range = High - Low
Range% = (High - Low) / Close × 100%
```

**意义**：
- 反映当日价格波动幅度
- 极差扩大 → 波动加剧，可能伴随重要消息
- 极差收窄 → 市场观望，可能即将变盘

**统计形式**：
- 20日平均极差：最近20天极差的平均值
- 最新极差：当天的极差值

---

### 2. 成交量加权平均价格 (VWAP)

**定义**：考虑了成交量权重的平均价格，反映真实的平均持仓成本

**公式**：
```
典型价格 TP = (High + Low + Close) / 3
VWAP = Σ(TP × Volume) / Σ(Volume)
```

**两种形式**：

| 形式 | 计算方式 | 用途 |
|------|---------|------|
| **累积 VWAP** | 从第一天到第N天累计计算 | 看长期资金成本 |
| **滚动 VWAP(20日)** | 最近20天滚动计算 | 看短期支撑/压力 |

**实战意义**：
- 价格 > VWAP：当前价格高于平均成本，强势
- 价格 < VWAP：当前价格低于平均成本，弱势
- 机构常用作算法交易的基准价格

---

### 3. 收益率 (Returns)

#### 3.1 简单收益率

**公式**：
```
R_t = (P_t - P_{t-1}) / P_{t-1} = P_t / P_{t-1} - 1
```

**特点**：直观易懂，但时间不可加

#### 3.2 对数收益率（Log Return）

**公式**：
```
r_t = ln(P_t / P_{t-1}) = ln(P_t) - ln(P_{t-1})
```

**核心优势**：**时间可加性**

**示例**（价格 100 → 110 → 100）：

| 计算方式 | 第一天 | 第二天 | 累计 |
|---------|--------|--------|------|
| 简单收益率 | +10% | -9.09% | +0.91% ❌ |
| 对数收益率 | +9.53% | -9.53% | 0% ✅ |

**何时近似相等**：单日涨跌幅 < 10% 时，两者几乎相等

#### 3.3 累计收益率

```
累计收益率 = (1 + R_1) × (1 + R_2) × ... × (1 + R_n) - 1
```

---

### 4. 波动率 (Volatility)

#### 4.1 核心概念

**定义**：收益率的标准差，衡量价格"上蹿下跳"的剧烈程度

**关键理解**：
- ❌ 不是"当天的波动"
- ✅ 是"最近N天收益率的波动程度"

#### 4.2 日波动率

**计算方式**：对最近N天的对数收益率求标准差

```
日波动率 = std([最近N天的对数收益率])
```

**不同时间窗口**：

| 类型 | 计算基础 | 反映周期 |
|------|---------|---------|
| **20日波动率** | std(最近20天对数收益率) | 短期（约1个月）|
| **60日波动率** | std(最近60天对数收益率) | 中期（约3个月）|
| **252日波动率** | std(最近252天对数收益率) | 长期（约1年）|

**注意**：前N-1天为NaN（数据不足无法计算）

#### 4.3 年化波动率

**公式**：
```
年化波动率 = 日波动率 × √252 ≈ 日波动率 × 15.87
```

**原理**：方差的时间可加性
- 日方差 = σ²_daily
- n日方差 = n × σ²_daily
- n日波动率 = √n × σ_daily

**对应关系**：
- `annual_vol_20` = 20日波动率 × √252（短期年化）
- `annual_vol_60` = 60日波动率 × √252（中期年化）

#### 4.4 月度波动率

**公式**：
```
月度波动率 = 日波动率 × √21 ≈ 日波动率 × 4.58
```

#### 4.5 波动率水平判断

| 年化波动率 | 风险等级 | 典型标的 |
|-----------|---------|---------|
| < 10% | 极低 | 货币基金、国债 |
| 10-20% | 稳健 | 沪深300ETF、蓝筹股 |
| 20-30% | 正常 | 普通股票 |
| 30-50% | 高 | 成长股、科技股 |
| > 50% | 极高 | 小盘股、加密货币 |

## 三、核心对比表

### 简单平均 vs VWAP

| 对比项 | 简单平均 | VWAP |
|--------|---------|------|
| 计算方式 | (High+Low+Close)/3 | 成交量加权 |
| 反映什么 | 价格水平 | **资金成本** |
| 谁在用 | 散户技术分析 | 机构算法交易 |

### 简单收益率 vs 对数收益率

| 对比项 | 简单收益率 | 对数收益率 |
|--------|-----------|-----------|
| 公式 | (P_t-P_{t-1})/P_{t-1} | ln(P_t/P_{t-1}) |
| 时间可加 | ❌ 否 | ✅ 是 |
| 使用场景 | 直观展示收益 | **波动率计算**、统计建模 |

### 不同窗口的波动率

| 窗口 | 反应速度 | 稳定性 | 适用场景 |
|------|---------|--------|---------|
| 20日 | 快 | 低（噪音大）| 短期风控、日内交易 |
| 60日 | 中等 | 中 | 中期配置、期权定价 |
| 252日 | 慢 | 高（最稳定）| 长期风险评估 |

## 四、Python 实现要点

### 1. 核心函数语法

```python
# 对数收益率
df['log_return'] = np.log(df['close'] / df['close'].shift(1))
# 或
df['log_return'] = np.log(df['close']).diff()

# 滚动窗口计算
.rolling(window=20).std()      # 20日标准差
.rolling(window=20).mean()     # 20日平均值

# f-string 动态列名
f'daily_vol_{window}'           # window=20 时结果为 'daily_vol_20'

# 取最新值
.iloc[-1]                       # 最后一行（最新数据）

# 年化换算
annual_vol = daily_vol * np.sqrt(252)
```

### 2. 完整计算流程

```
原始价格数据
    ↓
计算对数收益率
    ↓
滚动窗口求标准差 → 日波动率（20日/60日）
    ↓
乘以√252 → 年化波动率
乘以√21  → 月度波动率
```

## 五、常见误区澄清

### ❌ 误区 1：日波动率是当天的波动
**✅ 正确**：日波动率是**最近N天**日收益率的标准差，反映的是这N天的整体波动水平。

### ❌ 误区 2：年化波动率需要一年的数据
**✅ 正确**：只需要20天或60天数据即可**估算**年化波动率，公式是 `日波动率 × √252`。

### ❌ 误区 3：VWAP 就是简单平均价格
**✅ 正确**：VWAP 是**成交量加权**平均，成交多的地方权重更高，更能反映真实资金成本。

### ❌ 误区 4：波动率越高越好
**✅ 正确**：波动率高 = 机会多但风险大，波动率低 = 稳健但机会少。需要权衡（夏普比率 = 收益/波动）。

## 六、实战应用

### 1. 风险监控
```python
if annual_vol_20 > 30%:
    print("短期波动加剧，降低仓位！")
```

### 2. 趋势判断
```python
if annual_vol_20 > annual_vol_60 * 1.5:
    print("短期波动远高于中期，可能有大事件发生！")
```

### 3. 支撑压力（VWAP）
```python
if close > vwap_20:
    print("价格高于近期平均成本，趋势强势")
else:
    print("价格低于近期平均成本，趋势弱势")
```

### 4. 极差预警
```python
if current_range > mean_range * 2:
    print("今日波动异常扩大，关注是否有消息刺激！")
```
